A CLI `democraft` mantém o fluxo comum curto e preserva controles diretos de artefatos para CI e depuração.

## Uso

Instale a CLI e a API de autoria no projeto. O npm cria o executável local
`democraft`, então o `npx` usa a versão instalada:

```bash
npm install --save-dev @democraft/cli@beta @democraft/core@beta
npx democraft studio demo.ts
```

A CLI instala `@democraft/studio` como dependência interna. O comando inicia o
build de produção diretamente pelo Node; pnpm e um checkout do código-fonte do
Democraft não são necessários. Com pnpm ou Bun, use `pnpm exec democraft` ou
`bunx democraft`.

O caminho da demo é opcional quando o projeto contém exatamente um módulo convencional: `demo.ts`, `demo.tsx`, `src/demo.ts` ou `src/demo.tsx`.

```bash
npx democraft studio
npx democraft validate
npx democraft render -o demo.mp4
```

Se mais de um módulo convencional existir, passe o caminho explicitamente. A CLI nunca escolhe silenciosamente uma demo ambígua.

## Comandos

| Comando | Propósito |
|---------|-----------|
| `studio` | Captura e abre o Studio interativo. |
| `render` | Valida, captura, resolve a timeline e renderiza um MP4. |
| `validate` | Roda validação estática sem abrir o navegador. |
| `inspect` | Compila uma demo e imprime uma inspeção legível. |
| `capture` | Valida e cria artefatos reutilizáveis do navegador. |
| `targets` | Lista os contratos de alvo usados pela demo. |
| `timeline` | Resolve uma timeline a partir de um manifest explícito. |
| `preview` | *(Depreciado)* Escreve uma pré-visualização HTML autônoma. |

## Fluxos comuns

<Tabs items={["Studio", "Render headless", "Script do pacote"]}>
  <Tab value="Studio">
    ```bash
    npx democraft studio demo.ts
    ```

    O Studio captura quando necessário e reúne pré-visualização, edição da timeline e renderização em uma UI.
  </Tab>
  <Tab value="Render headless">
    ```bash
    npx democraft render demo.ts -o demo.mp4
    ```

    Este único comando executa validação, captura, resolução da timeline e renderização. O app-alvo precisa estar rodando durante a captura.
  </Tab>
  <Tab value="Script do pacote">

    ```json
    {
      "scripts": {
        "demo": "democraft studio demo.ts",
        "demo:render": "democraft render demo.ts -o demo.mp4"
      }
    }
    ```

    Scripts do pacote podem chamar diretamente o binário `democraft` local.
  </Tab>

## Opções

| Opção | Efeito |
|-------|--------|
| `-o, --output <path>` | Escreve o artefato no caminho escolhido. |
| `--json` | Imprime saída estruturada nos comandos de inspeção suportados. |
| `--headed` / `--headless` | Mostra ou oculta o navegador durante a captura. |
| `--output-dir <path>` | Usa um diretório de captura explícito. |
| `--fps <number>` | Sobrescreve os frames por segundo da timeline. |
| `--scale <number>` | Define a escala do render (padrão 1). |
| `--crf <number>` | Define a qualidade h264 (padrão 15, menor é melhor). |
| `--port <number>` | Define a porta do Studio (padrão 3000). |
| `--no-capture` | Apenas Studio: exige uma captura existente compatível. |

`--output-file` continua disponível como alias retrocompatível de `--output`.

## Modo avançado por artefatos

Use artefatos explícitos quando um job de CI separa captura, resolução da timeline e renderização:

```bash
npx democraft capture demo.ts
npx democraft timeline demo.ts \
  --manifest .democraft/runs/example/manifest.json \
  --output timeline.json
npx democraft render demo.ts \
  --manifest .democraft/runs/example/manifest.json \
  --timeline timeline.json \
  -o demo.mp4
```

`--manifest` e `--timeline` devem ser fornecidos juntos no render por artefatos. `--entry` continua como override avançado do entry do Remotion; componentes visuais normais são carregados automaticamente do `demo.ts`.

<Callout type="info">
Opções desconhecidas, valores ausentes, argumentos posicionais extras e módulos de demo ambíguos falham antes de iniciar captura ou renderização. Use `democraft render --help` para orientação específica do comando.
</Callout>
