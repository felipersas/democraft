Um componente customizado é um componente React que recebe um conjunto standard de props do renderizador e retorna JSX. Esta página percorre o contrato e um exemplo completo.

## O contrato

Todo componente de overlay recebe props do `OverlayLayer`. Existem dois contratos — um para captions, um para callouts.

### CaptionProps

<TypeTable
  type={{
    name: "CaptionProps",
    type: `{
  overlay: {
    id: string;
    stepId: string;
    sceneId: string;
    kind: "caption";
    text: string;
    fromFrame: number;
    durationInFrames: number;
    renderer?: string;
  };
  opacity: number;
}`,
  }}
/>

| Prop | Descrição |
|------|-----------|
| `overlay.text` | O texto da caption que o autor escreveu em `demo.ts`. |
| `opacity` | Um fade 0–1 computado a partir de `fromFrame`/`durationInFrames`. A camada rampa a entrada por 12 frames e a saída por 12 frames. Use para fazer o fade do seu componente. |
| `overlay.fromFrame` | O frame em que esta caption aparece. |
| `overlay.durationInFrames` | Quantos frames a caption fica visível. |

### CalloutProps

<TypeTable
  type={{
    name: "CalloutProps",
    type: `{
  overlay: {
    id: string;
    stepId: string;
    sceneId: string;
    kind: "callout";
    targetId: string;
    title: string;
    description?: string;
    fromFrame: number;
    durationInFrames: number;
    boundingBox?: BoundingBox;
    renderer?: string;
  };
  opacity: number;
  box: BoundingBox;
}`,
  }}
/>

| Prop | Descrição |
|------|-----------|
| `overlay.title` | Título obrigatório do callout. |
| `overlay.description` | Texto de descrição opcional. |
| `opacity` | Mesmo fade 0–1 das captions. |
| `box` | A bounding box do alvo na tela, **já transformada pela câmera**. Posicione seu callout relativo a isto. `BoundingBox = { x, y, width, height }` em coordenadas do stage. |

## Regras para um bom componente

<Steps>
  <Step title="Use `useCurrentFrame()` para animação">
    Todo componente roda dentro de uma composição Remotion. Importe
    `useCurrentFrame` de `"remotion"` e derive a animação do número do frame.
    Nunca use `setTimeout`, `requestAnimationFrame`, ou side effects — Remotion
    exige rendering puro e determinístico por frame.

    ```tsx
    import { useCurrentFrame, interpolate } from "remotion";
    ```
  </Step>

  <Step title="Clampe a interpolação">
    Sempre passe `{ extrapolateLeft: "clamp", extrapolateRight: "clamp" }` ao
    `interpolate` para que os valores não estorem fora do range de entrada:

    ```tsx
    const opacity = interpolate(frame, [0, 20], [0, 1], {
      extrapolateLeft: "clamp",
      extrapolateRight: "clamp",
    });
    ```
  </Step>

  <Step title="Honore a prop `opacity`">
    A camada computa um fade-in/fade-out para você. Multiplique sua própria
    opacity pela prop `opacity` para que o overlay faça o rampa corretamente:

    ```tsx
    style={{ opacity: myAnimationOpacity * props.opacity }}
    ```
  </Step>
</Steps>

## Exemplo completo: `remocn.pulse-callout`

Um callout que pulsa (escala para cima e para baixo) enquanto visível.

### 1. Crie o arquivo do componente

```tsx title="./components/pulse-callout.tsx"
"use client";

import { interpolate, useCurrentFrame } from "remotion";
import type { CalloutProps } from "@democraft/remotion";

export function PulseCallout({ overlay, opacity, box }: CalloutProps) {
  const frame = useCurrentFrame();

  // Pulso: escala oscila entre 0.97 e 1.03 a cada 30 frames
  const pulse = interpolate(
    frame % 30,
    [0, 15, 30],
    [0.97, 1.03, 0.97],
  );

  return (
    <div
      style={{
        position: "absolute",
        left: Math.min(box.x + box.width + 24, 1920 - 360),
        top: Math.max(42, box.y),
        width: 320,
        padding: 20,
        borderRadius: 12,
        backgroundColor: "rgba(99, 102, 241, 0.92)",
        color: "white",
        boxShadow: "0 16px 50px rgba(0,0,0,.3)",
        transform: `scale(${pulse})`,
        opacity,
      }}
    >
      <strong style={{ display: "block", fontSize: 24 }}>
        {overlay.title}
      </strong>
      {overlay.description ? (
        <p style={{ margin: "8px 0 0", fontSize: 18, lineHeight: 1.4 }}>
          {overlay.description}
        </p>
      ) : null}
    </div>
  );
}
```

### 2. Registre no seu entry

```ts title="./remotion-entry.ts"
import { defineVisualRegistry } from "@democraft/remotion";
import { PulseCallout } from "./components/pulse-callout";

const registry = defineVisualRegistry(
  { kind: "callout", id: "local.pulse-callout", component: PulseCallout },
);
```

### 3. Use numa demo

```ts title="demo.ts"
await demo.scene("result", async (scene) => {
  await scene.expectVisible("project-card");
  await scene.focus("project-card");
  await scene.callout("project-card", {
    title: "Done!",
    description: "Your project is ready.",
    renderer: "local.pulse-callout",
  });
  await scene.hold("2s");
});
```

### 4. Renderize

```bash
democraft render demo.ts --entry ./remotion-entry.ts --output-file out.mp4
```

## O que é exportado de `@democraft/remotion`

Para construir componentes, o pacote exporta:

| Export | Propósito |
|--------|-----------|
| `CaptionProps`, `CalloutProps` | Os contratos de props. |
| `VisualRegistry` | O tipo do registry. |
| `defineVisualRegistry(...entries)` | Constrói um registry customizado que estende os defaults. |
| `defaultVisualRegistry` | O registry built-in (todos `motion.*` + `remocn.*`). |
| `Caption`, `Callout`, `KineticCaption`, `GlassCallout` | Os componentes built-in (para estender ou referenciar). |
| `ProductDemoVideo`, `defaultProductDemoProps` | O componente de composição e suas props default. |
| `remocnAdapter(options?)` | A factory do adapter remocn. |

<Callout type="warn" title="Experimental">
Schemas de componentes (validação de props com zod) e presets de tema são planejados mas ainda não implementados. Hoje, a string `renderer` é opaque — o pipeline não valida se um renderer ID existe até o momento da renderização. Isso significa que um erro de digitação no nome do renderer cai silenciosamente no default sem erro.
</Callout>
