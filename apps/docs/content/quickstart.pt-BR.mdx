Este guia leva você de um checkout limpo a um MP4 renderizado usando o exemplo `demo-app` incluído. Você vai iniciar o app-alvo, capturá-lo e renderizar um vídeo.

## O que você precisa

- O workspace instalado e compilado — veja [instalação](./installation).
- Dois terminais.

<Callout type="info">
Esta página usa pnpm porque executa o exemplo dentro do workspace do
Democraft. Em uma aplicação que instalou `@democraft/cli`, use
`npx democraft studio`; o Studio de produção já é incluído pela CLI.
</Callout>

## Iniciar o app-alvo

O exemplo `demo-app` é um pequeno servidor HTTP que o Playwright captura. Ele precisa estar rodando **durante a captura** (mas não para pré-visualização/edição depois).

```bash title="Terminal 1"
pnpm --filter @democraft/example-demo-app start
```

Você deve ver `Demo app listening on http://localhost:4173`.

<Callout type="info">
O app-alvo só é necessário na **primeira** vez, para a captura. Depois disso, o Democraft reutiliza os arquivos capturados de `.democraft/` e o app não precisa rodar. Veja [captura única](./pipeline/capture-once).
</Callout>

## Abrir o Studio

O Studio é um app Next.js que pré-visualiza, edita e renderiza. Ele é aberto pela CLI, que grava um `meta.json` para que o Studio saiba onde estão a demo e suas capturas.

```bash title="Terminal 2"
pnpm exec democraft studio examples/demo-app/src/demo.ts
```

O primeiro lançamento roda o Playwright contra `:4173` e captura. Ao terminar, o Studio abre em `http://localhost:3000` mostrando a timeline capturada.

<Steps>
  1. **Navegue pela timeline.** O painel inferior mostra as faixas de câmera, overlay e cursor. O playhead e os controles de transporte ficam acima dela.
  2. **Pré-visualize.** Pressione <kbd>Espaço</kbd> para reproduzir. Use <kbd>←</kbd> / <kbd>→</kbd> para avançar frame a frame, <kbd>Home</kbd> / <kbd>End</kbd> para ir aos extremos.
  3. **Edite legendas.** Selecione um clipe de legenda na timeline para editar seu texto no inspetor — sem necessidade de re-captura.
  4. **Renderize.** Clique em **Render** no inspetor para produzir um MP4. O progresso é transmitido na fila de renderização.
</Steps>

## Renderizar pela CLI

Para renderizar sem o Studio, use o comando end-to-end `render` da CLI:

```bash
pnpm exec democraft render examples/demo-app/src/demo.ts -o out.mp4
```

O comando valida, captura, resolve a timeline e renderiza. Os artefatos intermediários continuam disponíveis em `.democraft/`, sem virarem argumentos obrigatórios da CLI.

<Tabs items={["Flags comuns", "Qualidade"]}>
  <Tab value="Flags comuns">
    ```bash
    -o, --output <path>    # onde escrever o MP4
    --fps <number>         # frames por segundo (timeline)
    --scale <number>       # escala de renderização (tente 2 para saída mais nítida)
    --crf <number>         # qualidade h264 (padrão 15, menor = melhor)
    --port <number>        # porta do studio (padrão 3000)
    --no-capture           # studio: reutiliza captura existente, falha se não houver
    ```
  </Tab>
  <Tab value="Qualidade">
    Para um vídeo mais nítido, aumente a escala de renderização e diminua o CRF:

    ```bash
    pnpm exec democraft render examples/demo-app/src/demo.ts \
      -o out.mp4 \
      --scale 2 \
      --crf 12
    ```
  </Tab>
</Tabs>

## Inspecionar e validar

Antes de capturar, você pode inspecionar ou validar estaticamente uma demo sem navegador:

```bash
pnpm exec democraft inspect examples/demo-app/src/demo.ts
pnpm exec democraft validate examples/demo-app/src/demo.ts
```

`inspect` imprime um resumo legível do IR compilado. `validate` roda os diagnósticos (`DC001`–`DC105`) e relata quaisquer problemas.
