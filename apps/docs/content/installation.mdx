## Use Democraft in a project

After the first npm release, install the CLI and the authoring API as development dependencies:

<Tabs items={["pnpm", "npm", "yarn"]}>
  <Tab value="pnpm">
    ```bash
    pnpm add -D @democraft/cli @democraft/core
    pnpm exec playwright install chromium
    ```
  </Tab>
  <Tab value="npm">
    ```bash
    npm install --save-dev @democraft/cli @democraft/core
    npx playwright install chromium
    ```
  </Tab>
  <Tab value="yarn">
    ```bash
    yarn add --dev @democraft/cli @democraft/core
    yarn playwright install chromium
    ```
  </Tab>
</Tabs>

Chromium is required for capture. Once your target app is running, validate or render a demo directly:

```bash
pnpm exec democraft validate demo.ts
pnpm exec democraft render demo.ts -o demo.mp4
```

<Callout type="warn" title="Pre-release repository">
The npm packages are not release-ready yet. Until the first release is completed, use the contributor setup below. In particular, the published CLI must gain a distributable Studio before `democraft studio` can work outside this monorepo.
</Callout>

## Contribute from source

Prerequisites: Node.js 20 or newer and pnpm 9 or newer.

```bash
git clone https://github.com/felipersas/democraft.git
cd democraft
corepack enable
pnpm install
pnpm build
pnpm exec playwright install chromium
```

Verify the workspace:

```bash
pnpm lint
pnpm typecheck
pnpm test
```

<Callout type="info">
Workspace packages resolve from their compiled `dist/` directories. Build once after installation; Turbo rebuilds changed dependencies on later workspace commands.
</Callout>

## Troubleshooting

<Callout type="warn" title="Workspace packages not found">
If source development reports `Cannot find module '@democraft/...'`, run `pnpm build` from the repository root.
</Callout>

<Callout type="warn" title="Playwright browser missing">
If capture reports `Executable doesn't exist`, run `pnpm exec playwright install chromium`.
</Callout>
