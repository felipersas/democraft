`defineDemo` declara e normaliza uma demo. Coloque targets e config diretamente na definição para o fluxo mais simples em um arquivo; separá-los é uma organização opcional para demos maiores.

## Assinatura

```ts
function defineDemo<TTargets extends Record<string, TargetInput>>(
  definition: DemoInput<TTargets>,
): DemoDefinition<DefinedTargets<TTargets>>;
```

## DemoDefinition

<TypeTable
  type={{
    name: "DemoDefinition",
    type: `{
  id: string;
  title: string;
  config?: DemoConfig;
  source: { baseUrl: string; initialPath?: string };
  targets: TTargets;
  run: (args: { demo: DemoCapture<keyof TTargets> }) => Promise<void> | void;
}`,
    description: "O formato de topo da demo.",
  }}
/>

| Campo | Tipo | Obrigatório | Descrição |
|-------|------|-------------|-----------|
| `id` | `string` | Sim | Identificador estável. Renomear invalida o cache de captura. |
| `title` | `string` | Sim | Nome legível por humanos. |
| `config` | `DemoConfig` | Não | Configuração reutilizável anexada a esta demo. |
| `source.baseUrl` | `string` | Sim | Origem do app-alvo. Precisa estar ativo durante a captura. |
| `source.initialPath` | `string` | Não | Caminho inicial. Padrão: o `goto` da primeira cena. |
| `authentication.profileId` | `string` | Não | Referência opaca a um perfil local. O estado do navegador não entra no código. |
| `targets` | `TTargets` | Sim | Contratos de elementos nomeados. As chaves literais se tornam os IDs aceitos pelos métodos de cena. Veja [locators](./locators). |
| `visuals` | `VisualMap` | Não | Definições locais e tipadas de visuais React compatíveis com Remotion. |
| `audioTracks` | `AudioTrackInput[]` | Não | Música, narração, efeitos ou ambiência apenas para apresentação. |
| `run` | `({ demo }) => Promise<void> \| void` | Sim | Conduz a captura via `demo.scene(...)`. |

Veja [áudio](../guides/audio) para timing e edição no Studio, e
[perfis de autenticação](../guides/authentication) para o ciclo completo da
sessão local.

## Exemplo mínimo

```ts title="demo.ts"
import { byTestId, defineDemo } from "@democraft/core";

export default defineDemo({
  id: "create-project-live",
  title: "Create a project live",
  source: {
    baseUrl: "http://localhost:4173",
    initialPath: "/dashboard",
  },
  targets: {
    dashboard: byTestId("dashboard"),
  },
  async run({ demo }) {
    await demo.scene("intro", async (scene) => {
      await scene.goto("/dashboard");
      await scene.establish("dashboard");
      await scene.caption("Create a workspace in seconds.");
      await scene.hold("5000ms");
    });
  },
});
```

## DemoCapture

O argumento `demo` para `run` é um `DemoCapture`. Ele tem um único método sobrecarregado:

<TypeTable
  type={{
    name: "DemoCapture.scene",
    type: `(id: string, run: (scene) => void) => Promise<void>`,
    description: "Define uma cena. Opcionalmente passe metadados como segundo argumento.",
  }}
/>

```ts
// sem metadados
await demo.scene("intro", async (scene) => { /* ... */ });

// com metadados
await demo.scene("intro", { purpose: "Apresentar o produto" }, async (scene) => { /* ... */ });
```

## defineConfig (opcional)

Para uma demo em um único arquivo, coloque a configuração diretamente na definição:

```ts
export default defineDemo({
  id: "save-profile",
  title: "Save profile",
  config: { fps: 30 },
  source: { baseUrl: "http://localhost:3000" },
  targets: {},
  async run() {},
});
```

`defineConfig` continua disponível quando várias demos compartilham intencionalmente as mesmas configurações.
